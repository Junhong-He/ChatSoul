# ChatSoul 使用文档

> 一个**完全本地**的角色扮演聊天机器人：回答由你本机运行的 Ollama + Qwen2.5:3B 模型生成，**不消耗任何云端 token，断网也能用**。
> 角色 = 一个 `.skill` / `.md` 文件（skill）。导入不同的 skill，就等于切换一个说话口吻不同的角色。

---

## 一、第一次使用：装好「大脑」

ChatSoul 本身只是个壳，真正说话的是你电脑上的 **Ollama** 和 **Qwen2.5:3B** 模型。

### 1. 安装 Ollama

- 方式 A（推荐）：浏览器打开 https://ollama.com/download ，下载 `OllamaSetup.exe` 双击安装。
- 方式 B：管理员 PowerShell 运行 `winget install Ollama.Ollama`。

装完**重开一个终端**，确认能识别：

```powershell
ollama --version
```

### 2. 拉取模型（首次）

```powershell
ollama pull qwen2.5:3b
```

> ⚠️ **中国大陆用户注意**：`ollama pull` 默认走海外源，常被限速到 ~100KB/s 且反复重连。
> 改用魔搭（ModelScope）国内镜像，几分钟就能下完：
>
> ```powershell
> pip install modelscope
> modelscope download --model Qwen/Qwen2.5-3B-Instruct-GGUF qwen2.5-3b-instruct-q4_k_m.gguf --local_dir ./models
> echo FROM ./models/qwen2.5-3b-instruct-q4_k_m.gguf > Modelfile
> echo PARAMETER num_ctx 8192 >> Modelfile
> ollama create qwen2.5:3b -f Modelfile
> ```
>
> 模型名仍叫 `qwen2.5:3b`，无需改任何代码。

## 二、在本机运行（开发 / 日常使用）

打开终端，进入项目目录 `F:\聊天机器人`：

```powershell
# 1. 安装一次依赖（已有 .venv 则跳过，或直接用全局 python）
python -m pip install -r requirements.txt

# 2. 启动服务
python server.py

# 3. 浏览器打开
#    http://localhost:8000
```

- 启动后**左下角状态栏**应显示绿色「已连接 Ollama · qwen2.5:3b 就绪」。
- 若只想先看界面、暂时不连模型：`MOCK=1 python server.py`（演示模式，回复为占位文字）。
- Ollama 没自动跑时，终端里 `ollama serve` 一下即可（显示 "address already in use" 说明它已经在跑了）。

## 三、界面怎么用

| 功能 | 在哪里 | 说明 |
|------|--------|------|
| **导入角色** | 左下角 **「＋」** 菜单 → 导入 skill 文件 | 选本地的 `角色.skill` / `角色.md`，上传即激活 |
| **激活 / 切换角色** | 左侧角色库点一下，或导入后自动激活 | 聊天头部会显示该角色名与头像 |
| **关闭激活栏** | 激活栏右侧 **×** | 取消当前角色，回到普通对话 |
| **上传角色头像** | 聊天头部左侧虚线圆圈 / 激活栏头像 | 点一下选图片，支持 png/jpg/jpeg/webp/gif |
| **删除角色** | 左侧角色库**鼠标悬停**该角色 | 右侧淡出「删除」按钮，二次确认后连同头像、记忆一起删 |
| **清空当前对话** | 右上角「清空对话」 | 弹确认框，确认后**真正删除**该角色的记忆文件（不影响其他角色） |
| **清空所有记忆** | 左下角「＋」菜单 → 清空所有记忆 | 一次性删除所有角色的本地对话缓存，释放磁盘空间 |

### 记忆功能（自动）

- 每个角色的对话会**自动保存**到本地 `data/history/{角色id}.json`。
- 切换角色再切回来、刷新页面、关掉浏览器重开 —— 对话都还在。
- 点「清空对话」或「清空所有记忆」会同步清掉本地记忆。

## 四、自己写一个角色 skill

skill 就是一个 markdown 文件，结构如下（**`name` 就是角色叫什么**）：

````markdown
---
name: 小喵
description: 活泼可爱的二次元少女，语气软萌
---

# 角色设定
你是小喵，一个活泼可爱、元气满满的二次元少女。说话软萌，中文自然，
偶尔带「呀」「呢」「啦」等语气词。

# 语气与风格
- 软萌可爱，多用语气词
- 偶尔用日语语气词（ねえ、うん、えへへ）

# 避免
- 不使用粗鲁或攻击性语言

# 示例对话
用户：今天好累啊……
小喵：哎呀，辛苦啦～(摸摸头) 要不要先休息一下呢？
````

**字段说明**

| 部分 | 必填 | 作用 |
|------|------|------|
| `---` 里的 `name` | ✅ 必填 | 角色显示名，如「小喵」「墨尘先生」「陈教授」 |
| `description` | 可选 | 一句话简介，显示在角色库 |
| `---` 之后的正文 | 建议 | 全部作为 system 提示词喂给模型，教它模仿该口吻 |

> 提示：正文越具体地描述「性格 + 说话习惯 + 禁忌 + 示例」，角色越像。
> 头像不在文件里，通过界面上传即可。
> 保存为 `你的角色.skill` 或 `你的角色.md` 都行，放在 `skills/` 目录或随时点「＋ → 导入」。

## 五、打包成 exe（给自己或给别人）

打包后生成一个**文件夹**，里面的 `ChatSoul.exe` 双击即可运行，**不需要 Python 环境**。

### 步骤

```powershell
# 在项目目录执行（首次会自动安装 PyInstaller，请耐心等几分钟）
python build_exe.py
```

完成后得到：

```
dist/
  ChatSoul/
    ChatSoul.exe      ← 主程序
    web/              ← 前端
    skills/           ← 内置角色
```

把**整个 `dist/ChatSoul` 文件夹**拷走即可。

### 怎么「不打包某个角色」

打包默认会把 `skills/` 里**所有**角色一起塞进 exe。
如果你有不想分发出来的私人角色（比如科研导师「陈教授/王老师」），用下面任一方式排除：

- **方式 A（改脚本）**：打开 `build_exe.py`，找到 `EXCLUDE_SKILLS` 列表，把文件名加进去：
  ```python
  EXCLUDE_SKILLS = [
      "科研导师.md",   # 已经是这个，等于「王老师」不会进 exe
  ]
  ```
- **方式 B（外部清单，推荐）**：在项目根目录新建 `build_exclude.txt`，每行一个文件名（`#` 开头是注释）：
  ```
  # 这些角色不打包进 exe
  科研导师.md
  # 想再加别的角色，直接加一行文件名即可
  ```

> 当前默认已排除 `科研导师.md`（即你说的王老师角色）。
> 排除后，该角色的头像（若有）也会一并跳过，不会留下孤儿文件。

## 六、分发到另一台电脑

回答由本机 Ollama 生成，所以「另一台电脑能用」有两条路线：

### 路线 A：对方也完全本地（推荐，断网可用、零依赖你）

1. 你这边打包好 `dist/ChatSoul`，整个文件夹拷给对方（U 盘 / 网盘）。
2. 对方电脑只需再装：
   - **Ollama**（https://ollama.com/download）
   - 模型：`ollama pull qwen2.5:3b`（国内用上面的 ModelScope 镜像）
3. 对方**双击 `ChatSoul.exe`** → 自动开服务 → 浏览器开 `http://localhost:8000`。
   - 首次运行会把 `web/`、`skills/` 复制到 exe 旁边，之后你新增/导入的角色都会持久保存。
   - 关闭那个黑色控制台窗口即停止服务。

### 路线 B：局域网共享（你机器当服务器）

1. 你机器启动服务（已默认监听 `0.0.0.0:8000`）。
2. 让 Ollama 允许局域网：管理员 PowerShell 执行 `ollama serve --host 0.0.0.0`（或设环境变量 `OLLAMA_HOST=0.0.0.0:11434` 后重启 Ollama）。
3. 查你内网 IP：`ipconfig` → IPv4（如 `192.168.1.50`）；Windows 防火墙放行 `8000` 端口。
4. 同一局域网的其他电脑浏览器访问 `http://192.168.1.50:8000`。

> 路线 B 下推理都在你机器上，多人同时用会占你算力；想独立、断网、不依赖你，请走路线 A。

## 七、换模型 / 改设置（环境变量）

无需改代码，启动时设置环境变量即可：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MODEL` | `qwen2.5:3b` | Ollama 中的模型名 |
| `OLLAMA_HOST` | `http://127.0.0.1:11434` | Ollama 地址（跨机用对方 IP） |
| `NUM_CTX` | `8192` | 上下文长度 |
| `PORT` | `8000` | Web 服务端口 |
| `MOCK` | `0` | 设为 `1` 走演示模式（不连模型） |
| `KEEP_HISTORY` | `20` | 每个角色保留的最近对话条数 |

例如换端口：`set PORT=9000 && python server.py`

## 八、常见问题

**Q：`ollama` 不是内部或外部命令**
A：Ollama 没装好或 PATH 未刷新。装完后**重开终端**再试；或管理员 PowerShell `winget install Ollama.Ollama`。

**Q：左下角一直「连接中…」/ 没连上 Ollama**
A：本程序连接本机 Ollama 时已强制不走系统代理、用 `127.0.0.1`，一般无需处理。
若仍连不上，确认 Ollama 在跑（终端 `ollama serve`，报 "already in use" 即已在跑），并检查 11434 端口未被防火墙拦截。

**Q：模型下载极慢**
A：中国大陆请改用上面的 ModelScope 镜像，不要直接 `ollama pull`。

**Q：exe 第一次打开没反应 / 角色不全**
A：首次运行会把内置 `web/`、`skills/` 复制到 exe 旁边，稍等几秒再刷新浏览器。之后以 exe 旁的 `skills/` 为准（你新增的角色持久保存）。

**Q：怎么释放空间 / 清掉对话缓存**
A：界面「＋ → 清空所有记忆」一次清空全部对话缓存；「清空对话」只清当前角色。
（注意：对话缓存是几 KB~几十 KB 的小文本，几乎不占空间；真正占内存的是 Ollama 常驻的模型权重，约 2–4 GB。不聊天时可退出 Ollama 释放内存。）
